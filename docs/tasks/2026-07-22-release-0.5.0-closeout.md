## Objective

Publish `v0.5.0` as the next pre-1.0 minor release after `v0.4.0`, carrying the accepted
parameterless provider/model flows, custom-provider discovery and activation, and explicit local
image attachments through the standard tag-driven release workflow.

## Version Decision

`v0.5.0` is a minor release because it adds user-visible provider registration/model discovery
and multimodal attachment capability, and introduces pre-1.0 public API changes. This does not
claim v1.0 readiness: REL-002 remains NO-GO.

## Release Boundary

- Includes parameterless `/model` and `/connect`, custom compatible-provider registration,
  discovered-model activation, and explicit `/attach` / `/attachments` / `/detach` image flows.
- Includes safe capability fail-closed behavior, exact-path approval, decoding/size/pixel limits,
  digest revalidation, and path-safe history summaries.
- The OpenAI-compatible image flow has maintainer terminal evidence. Anthropic-compatible image
  wire mapping is covered by deterministic fixtures, but a live provider check remains unavailable
  in the maintainer environment; the release must not claim that live check passed.
