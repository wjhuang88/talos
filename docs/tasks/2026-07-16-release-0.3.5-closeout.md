# Release 0.3.5 Closeout

**Status**: Complete — validated tag pushed; CI tracking waived (2026-07-16)

## Objective

Publish `v0.3.5` as the next pre-1.0 patch release after `v0.3.4`, carrying the accumulated runtime,
session, dashboard, TUI, scheduler, durable-session, and model-private file-edit improvements on
`main` through the repository-standard tag-driven release workflow.

## Scope

- Synchronize the workspace package version and every internal path dependency to `0.3.5`.
- Update README release references and the Windows installer-trial default tag to `v0.3.5`.
- Run `./scripts/release_preflight.sh v0.3.5` with the pinned Rust 1.97.0 toolchain.
- Create and push one immutable annotated `v0.3.5` tag after preflight passes.
- Let `.github/workflows/release.yml` own the asynchronous platform builds, checksums, and GitHub
  Release publication after the tag is pushed.
- Record the validated release handoff here and on the Board. Per maintainer direction, this task
  does not wait for or track the asynchronous CI build result.

## Non-Goals

- No crates.io publication.
- No tag rewrite, force-push, or reuse after failure.
- No `v1.0.0` readiness claim; REL-002 remains NO-GO.
- No new runtime, permission, sandbox, session-format, storage-default, or dependency behavior in
  the version-bump commit.
- No manual release asset upload while the tag-driven workflow is operating normally.

## Release Content Boundary

GitHub Release notes are generated from commits after `v0.3.4`. The release includes already
reviewed work on `main`, notably runtime event convergence, scheduled follow-ups, embedded durable
sessions, rendered loopback dashboard pages, TUI composer history, audit/decision closeouts, and
I134 model-private snapshot-anchored file edits. MEM-009/ADR-046 is documentation and future
architecture only; its implementation is not activated by this release.

## Validation Plan

- `cargo metadata --locked --no-deps --format-version 1` reports one Talos version: `0.3.5`.
- `cargo fmt --all -- --check`.
- `cargo check --workspace --locked`.
- `cargo clippy --workspace --locked -- -D warnings`.
- `cargo test --workspace --locked`.
- `./scripts/release_preflight.sh v0.3.5`.
- `scripts/validate_project_governance.sh .`.
- `git diff --check`.
- Staged release diff reviewed; no credentials or unrelated changes.
- Annotated tag and `origin/main` point to the release commit.
- Annotated `v0.3.5` tag is pushed to `origin`, handing publication to the tag-driven GitHub
  workflow; CI completion is intentionally not tracked in this task.

## Failure And Recovery

- Preflight failure: do not create the tag; fix on `main` and rerun.
- Workflow failure after tag push: keep `v0.3.5` immutable, record the failure, fix the source, and
  publish a new patch version such as `v0.3.6`.
- External GitHub outage: record the pushed commit/tag and workflow blockage without claiming a
  completed GitHub Release.
