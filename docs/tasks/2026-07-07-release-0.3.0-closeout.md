# Release 0.3.0 Closeout

**Status**: Ready for Tag (validation passed; tag/push pending)

## Objective

Publish `v0.3.0` as the next pre-1.0 stable release line. This is a minor-version bump from
`v0.2.2` to capture the accumulated provider/runtime hardening, model/provider UX work, MCP remote
transport support, and market-trial readiness planning.

## Scope

- Bump workspace and internal crate dependency versions from `0.2.2` to `0.3.0`.
- Update README release boundary text in English and Chinese.
- Validate the workspace before tagging.
- Create and push the `v0.3.0` tag after validation passes.

## Non-Goals

- No crates.io publish.
- No manual GitHub Release creation; the tag-driven release workflow owns release assets and
  GitHub Release body generation.
- No permission-default change, release-asset hand editing, or installer path redesign.

## Release Contents Summary

This release includes changes since `v0.2.2`, including:

- OpenAI-compatible streaming provider hardening for missing tool-call ids and `[DONE]` after
  streamed tool calls.
- Agent turn-loop guards for malformed `ToolUse` event sequences and duplicate tool-call ids.
- MCP remote HTTP/SSE and Streamable HTTP support.
- Model/provider UX hardening: standard-provider no-URL connect flow, large model-list work, and
  provider metadata/protocol handling.
- Runtime/permission/validation planning and closeout records for the market-trial readiness path.
