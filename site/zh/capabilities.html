内置工具

Talos 内置了覆盖常见编码 Agent 工作流的工具。默认提示词会要求模型优先使用 内置工具，只有当原生工具无法覆盖任务时才回退到 shell 命令。

斜杠命令

在 TUI 中输入 / 打开命令菜单。继续输入可筛选命令，使用 Up / Down 移动选项，按 Enter 或 Tab 完成。Backspace 用于编辑筛选内容，Esc 关闭菜单且不清空输入区。 使用 /help 查看当前会话可用命令。Skill 命令在 inline 模式也可使用。

命令 说明
/help显示可用命令
/quit, /exit退出 Talos
/status显示会话信息（模型、token 用量）
/plugins列出已观察到的工具来源与 MCP server 状态
/skills列出可用运行时 skills 及激活状态
/skills activate <name>激活一个 Skill 主体，用于后续 provider 请求
/skills reference <path>为激活 Skill 加载有界引用文件
/copy last复制上一条助手消息到剪贴板
/copy all复制完整对话记录到剪贴板
/export <path>导出对话记录到文件（需权限批准）
/new开始新会话（保留旧会话）
/resume列出可恢复工作区会话；/resume <N> 按序号选择
/fork分叉当前会话（克隆历史到子会话）
/delete打开会话选择器（排除当前会话）；选中后删除
/model打开模型选择器，在运行时浏览并切换模型

运行时 Skills

Talos 会在会话启动时发现 SKILL.md 文件，并在第一次模型回合前将 Level 0 元数据（skill 名称、描述和触发词）注入系统提示词。

Skill 搜索路径（按优先级）：

  1. 当前工作区的 .talos/skills/
  2. ~/.talos/skills/
  3. 向上至 Git 根目录过程中继承的父级 .talos/skills/ 目录

在 TUI 或 inline 模式使用 /skills 可查看运行时发现的 skills。 使用 /skills activate <name> 可显式加载一个 Skill 主体到 provider 上下文用于后续回合。Skill 激活后，可使用 /skills reference <relative-path> 从该 Skill 目录加载有界引用文件。

渐进式上下文：激活的 Skill 主体和引用仅加入 provider 上下文。Talos 不会将完整内容打印到滚动历史、命令输出或 transcript 历史。引用路径必须保持在当前激活 Skill 目录内。

MCP 工具

在 ~/.talos/config.toml 中配置本地 stdio MCP servers： 

[[mcp.servers]]
name = "filesystem"
transport = "stdio"
command = "/path/to/mcp-server"
args = ["/path/to/workspace"]
env = {}

Talos 会在 TUI、print、inline、interactive 和 RPC 模式首个模型回合前启动 已配置 server 并发现其工具。工具名称采用 mcp:<server>:<tool> 形式。只读标注会被保留；其他 MCP 工具会进入常规批准流程， 在无法交互批准时默认拒绝。启动失败会被报告但不会中止会话， 每个 MCP 请求也有超时上限。可在 TUI 中使用 /plugins 查看启动连接快照和已观察到的工具来源。

稳定性：MCP 工具集合在会话生命周期内固定， 以保持模型可见工具定义和提示词缓存前缀稳定。修改 MCP 配置后请重启会话。 当前仅支持本地 stdio transport。

在 Rust 中嵌入 Talos

Rust 应用可以依赖 talos-runtime crate，在不链接 Talos CLI 或 TUI crates 的情况下嵌入核心 Agent 循环。 当前 pre-1.0 facade 提供 RuntimeBuilder 和 RuntimeHandle，用于 provider / tool 注入、类型化事件流、中断、关闭和显式请求预览。

已注册工具默认会经过权限包装。在 headless 嵌入中，未解决的 Ask 决策会被拒绝，除非 embedder 提供更窄的 allow-list 规则。

pre-1.0 表面：这还不是稳定的 1.0 SDK 承诺。 当前公开嵌入表面是 talos-runtime 以及它从 talos-core 重新导出的协议与 trait 类型。更底层的 talos-agent 构造器仍属于实现表面，除非文档另有说明。 详见 ADR-024 和用于 v1.0 自举门槛的 路线图。
