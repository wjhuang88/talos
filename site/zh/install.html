安装

请选择与你平台匹配的安装方式。对于 v0.2.0，官方安装器和 release 压缩包是受支持的二进制入口。crates.io 上的 Cargo install 形态已经规划，但尚未发布。

下载 release

在 macOS 或 Linux 上安装最新 release：

curl -fsSL https://raw.githubusercontent.com/wjhuang88/talos/main/install/install.sh | sh

在 PowerShell 中安装最新 Windows x86_64 release：

iex (irm https://raw.githubusercontent.com/wjhuang88/talos/main/install/install.ps1)

安装器位于仓库的 install/ 下，因为它们是面向用户的 release 入口。开发和治理脚本位于 scripts/； 在 pre-1.0 安装器目录整理之后，旧的 scripts/install.* 路径被有意不再保留。

或下载压缩包

从 GitHub Releases 下载你平台对应的压缩包，然后解压： 

tar -xzf talos-aarch64-darwin.tar.gz
chmod +x talos
./talos --help

已发布的压缩包名称

平台 压缩包
Linux x86_64 talos-x86_64-linux.tar.gz
Linux ARM64 talos-aarch64-linux.tar.gz
macOS Intel talos-x86_64-darwin.tar.gz
macOS Apple Silicon talos-aarch64-darwin.tar.gz
Windows x86_64 talos-x86_64-windows.zip

注意：Windows ARM64 产物尚未发布。 在发布可用产物之前，Windows 安装脚本也会拒绝在 ARM64 上运行。

首次运行设置

当你在未配置模型时启动 Talos，TUI 会打开模型选择器而不是直接失败。 选择一个模型即可开始。如果 provider 需要凭据，Talos 会显示 API key 的设置说明。

在 CI 或非交互环境中跳过向导：

talos --no-init -p "summarize this repo"

配置管理

无需手动编辑 TOML 即可查看和修改配置。密钥在所有显示面都会脱敏； 完整的密钥脱敏边界见 安全页面。 

talos --config-list                          # print all settings (secrets masked)
talos --config-get model                     # get a single value
talos --config-set model=claude-sonnet-4-20250514  # set and persist
talos --config-set providers.anthropic.api_key_env=ANTHROPIC_API_KEY

从源码构建

环境要求：

  • Rust 1.95 或更新版本
  • Cargo
cargo build --release -p talos-cli
./target/release/talos --help

如需在本地构建全部 release 产物：

./build.sh

多平台构建会将压缩包和校验和写入 dist/。

Cargo install 状态： cargo install talos-cli --bin talos 是计划中的 crates.io 二进制安装形态，但仍被发布门禁阻塞。本地源码测试可使用 cargo install --path crates/talos-cli --bin talos --locked。

验证安装

安装后，运行一次 print 模式单次提示，确认二进制端到端可用： 

talos -p --mock "/mock-request summarize this repository"

mock provider 会返回确定性的本地响应，无需触发网络 provider。 这也是 TUI 冒烟测试在连接真实模型前验证运行时路径所使用的命令。
