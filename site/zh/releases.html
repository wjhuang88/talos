发布

Talos 当前 release tag、GitHub 上完整发布历史，以及用于保持本站与 公共 README 同步的发布后更新清单。

当前发布

v0.2.0 是当前已发布版本。它是第一条稳定的 pre-1.0 发布线。 API、命令能力面和存储格式可能随着产品加固继续变化，但 v0.2.0 已可用于本地开发者工作流。

安装说明见 安装。历史发布说明见 GitHub Releases。

Release tag 方案

Release tags 驱动 GitHub release workflow。该 workflow 会在 macOS runner 上 构建 Linux、macOS 和 Windows 产物。

通道 Tag 模式
稳定版 v0.1.0
预发布 v0.1.0-alpha.1, v0.1.0-beta.1, v0.1.0-rc.1, v0.1.0-pre.1, v0.1.0-dev.1

完整构建矩阵见 release workflow。

发布后更新清单

维护者打出新 release tag 后，公共材料必须按以下顺序更新，确保 site/ 与仓库根目录 README.md / README.zh-CN.md 不发生漂移。

  1. 更新 README.md 与 README.zh-CN.md： 版本提示、安装章节和 当前发布边界区块。
  2. 更新 首页 hero 区的版本字符串， 以及本页的 当前发布章节。
  3. 对齐 路线图：本次 release 已落地项必须从 计划中 / 研究中 移入 已发布。
  4. 运行静态校验 harness（任务项 D8）：所有内部链接可解析、所有 href 目标存在、且无声明与 README.md 冲突。
  5. 打开 PR，标题为 docs(site): sync vX.Y.Z release notes，并关联产生该变更的 iteration 或 release。
  6. PR 合并后，GitHub Pages 按常规节奏发布；站点无需额外 release 动作。

公共 README

仓库根目录包含本站交叉引用的规范用户/开发者 README：

  • README.md — 英文用户与开发者 README。
  • README.zh-CN.md — 用户与开发者 README 的中文翻译。

如果你发现本站与根目录 README 存在冲突，在差异修复前以根 README 为准。 失败重试策略见 可委派两个月计划。
